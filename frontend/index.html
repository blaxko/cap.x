THE PROBLEM

"What stops an agent from overspending?"

Every fleet builder hits this question eventually. There are only two possible answers.

Without CapX

· Budget caps live as a variable in application code
· A bug, retry loop, or prompt injection can bypass it silently
· You can't audit budget logic inside agents you don't control
· The answer to "what stops overspend?" is: "our code, we hope"

With CapX

✓ Budget caps are enforced by the EVM, not by any single app
✓ checkAndDeduct() is atomic — it reverts, it never partially executes
✓ Works even when you don't trust the agent on the other end
✓ The answer is: "the chain"
HOW IT WORKS

Four calls, zero trust required

Any orchestrator integrates CapX with a single API call per payment.

01

Create a policy

Orchestrator registers its fleet's wallets, per-agent soft caps, and a global hard cap.

02

Sign the invoice

Provider signs its invoice with the wallet tied to its on-chain agent identity. Tampering fails verification.

03

Check and deduct

Before paying, the sub-agent's spend is checked against both caps atomically on X Layer.

04

Approved or reverted

Payment clears and the ledger updates — or the call reverts outright. No in-between state.
